Pytest:

Pytest is a testing framework that lets users build test scripts in Python. It aids in the creation of easy and scalable test cases for databases, APIs, and user interfaces. Pytest is mostly used for writing API tests. It aids with the development of tests ranging from simple unit tests to large functional tests.

Testing with Pytest:

Pytest has many advantages in terms of testing. It is easy and simple syntax. Therefore, it’s easy to start. It can run tests in parallel. It can run a specific test or a subset of tests. It detects tests automatically and skip tests.

Creating simple pages and testing:

‘Simple page’ is a page that we can put on web. It uses a feature in flask called ‘blueprint’. A blueprint is a package of controller where controller works as a module/ plugin. So we start to making a blueprint. Therefore, I will make a ‘init_.py ‘file in the folder templates of ’simple pages’ folder. In the flask app, I will have framed of that blueprint which has route, controller in it. ‘route’ sets the variable called ‘page’. Due to having this variable, I can make a html file called ‘page’. This ‘page’ would be my html file. For Instance, i can make a page1.html file to put my web content in the same folder where I have ‘init_py‘ file.


image(b) Example of int_py file in my flask app


To creat a simple page, below process is explained with steps and examples.

Step 1.
  • I have docker_flask file in my local computer . This file contains hierarchical framework for web hosting work . I open this project file in Pycharm software. Then, I can also run docker container for the docker file of this project and get a successful running logs.
  • Next, I go to the ‘app’ folder of my project and then go in ‘simple pages’ sub-folder. I have a ‘templates’ folder there. I press right-click on the folder and creat a ‘page2.html’ file.

  • I have a ‘simple_pages_test.py’ file in my test folder of the project. I use this file for my pytest. Therefore, I will add below-mentioned code for testing my ‘page2’.
    Code: def test_request_page2(client)

    response = client.get("/page2")
    assert response.status_code == 200
    assert b"Page 2" in response.data

  • I go to the terminal. In the service option from menubar, I can see my running services for ‘/docekr_flask-flsk-1’ file. I right-click on that file and then create a terminal, if I don’t have an open terminal at this place.
Step 5.
  • I can write the below-mentioned command in the terminal to run pytest.
    Command: pytest

The purpose of files in the project:


Project flask(root)\ >>
  • docker-compose.yml :- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku
  • Dockerfile:- File used to create an image to run in a container that runs the program
  • readme.md :- It is likely the first file a reader encounters when they browse a directory that contains your code. In this way, it acts as a landing page for the directory.
  • setup.py :- It is a python file, the presence of which is an indication that the module/package you are about to install has likely been packaged and distributed with Distutils, which is the standard for distributing Python Modules.
  • heroku.yml:-we can add a file as our webhost. here, heroku is my webhost. I can mention in this file docker container and a web for realising an image.
Project flask \ >> ‘app’(Root folder)\ >> simple pages\ >> templates\
  • This is where I store my html templates i.e. index.html, layout.html
  • page.html :- ‘page’ is a name of html file. we can give any name to this html file with .html extension. This files writes the content of web in respect with html syntax.
Project flask \ >> ‘app’(Root folder)\ >> static\
  • style.css: it helps to give a style to html file.
  • image.png: we can put images for our project that we need in our web.
  • scripts.js: it has java-script and helps to manipulate the code regarding our web content.
Project flask \ >> ‘app’(Root folder)\ >> templates
  • I can store base html file also here as base.html where i can create by base structure of webpage.
  • run.py :- contains the actual python code that will import the app and start the development server.
  • __init__.py :- initializes your application creating a Flask app instance.
Project flask \ >> test (Root folder)\
  • This folder,we can add pytest files here. we can add as additional files according to requirement and remove pytest files.
Project flask \ >> .github (Root folder)\ >> workflows\
  • This folder helps to give a direction of workflow to GitHub account
  • test.yml:- It is to create configuration files. For instance, it gives a docker image, account ,host website and our account information connecting to GitHub repository.
