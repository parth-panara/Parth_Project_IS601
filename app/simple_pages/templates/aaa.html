A test is a procedure that exercises a behavior to determine if the behavior functions correctly.

For test, there is a simple and powerful pattern has been set up that known as a : Arrange-Act-Assert

Arrange

It means prepare inputs, targets all the operations like need of object, log in web etc.

Act

Execution is being taken into the programming code syntax. For Instance, it could be for function, method, web page.

Assert

It means to add the result that we expect as outcome from program. So we can verify/ test the response from it. By doing assertions, we can check string or numeric values.

Why is testing important?

  • As well as other sorts of testing,AAA testing is critical to ensuring that our software is up and operating without any problems.
  • The major purpose of unit testing is to find bugs and other inconsistencies in the application while it is still in development so that these issues can be caught before the software is released into production.
  • The positive side of AAA testing is to concentrate each test on a particular activity. It distinguishes between the configuration of the System Under Test and the assertions made on it after the activity.
  • The second thing is that it is well-organized and tidy, which helps to unify and explain the structure of tests, making them easier to comprehend and manage.
