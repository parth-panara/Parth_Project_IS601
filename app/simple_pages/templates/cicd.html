Continuous Integration and Deployment


Github actions:

Automate, customize, and execute your software development workflows right in your repository with GitHub Actions. We can discover, create, and share actions to perform any job we'd like, including CI/CD, and combine actions in a completely customized workflow.


Project setup for GitHub actions:

Step 1: Under my repository name, click Pull requests.

Step 2: In the list of pull requests, click the pull request you'd like to review.

Step 3: On the pull request, click +/- Files changed.

Step 4: Review the changes in the pull request, and optionally, comment on specific lines.

Step 5: Above the changed code, click Review changes.

Step 6: Type a comment summarizing your feedback on the proposed changes.

Step 7: Select Approve to approve merging the changes proposed in the pull request.

Step 8: Click Submit review.


Deploy master to a production server:

Prerequisite

1. I am assuming one master branch as a default branch in my GitHub repository to explain this project.
2. Here, I use my docker software for my image and container and heroku software to deploy my web. I have a credentials of both accounts( username, password, email)

The steps in the process are as follows:

Step 1: I create a .yml file in my project. I use pycharm software to do it. ‘.yml’ file contains a code of configure settings for my docker, GitHub and heroku platform. it draws the path of CI/CD test for code and successful connection with docker and heroku. I referred the reference code for .yml file from my professor’s flask application.

Step 2: in pycharm, I can create a new branch and then i can edit, add, and delete the code according to requirement for my application work. For Instance, I give a branch name ‘testingCI/CD’ .(In this case my main/ master branch in PyCharm would remain without interruption)

Step 3: I push that new branch with adding a commit ‘Updated and testing CI/CD’ through the commit process to GitHub.

Step 4. Now, I make sure that I have two branches in my GitHub repository. one is ‘master’ named and another one is ‘testing CI/CD’.

Step 5: I add my docker and heroku app credentials in my secret option of Action tab of GitHub repository. (Note: This credentials coordinates with my project’s ‘yml. ’ file configuration)

Step 6: With the help of above steps, I have set an automated deployment process for my work. Now, I merge my ‘testingCI/CD’ branch to the master branch. To do it, I select a pull request option in my GitHub repository to merge the branches. In this process, GitHub works in three steps like test, built docker container , and deploy heroku. After successful completion of these three steps I get my updated master branch.

Step 7: On completion of step-6, my master branch is now connected with the heroku app server. Heroku application is ready to host my webapp/webpage with the help of master branch of GitHub. Thus, I get a successful deployment of my master branch to the production server.


Deploy an image of my project to Dockerhub when master is updated:

Elaborating the step 6, I create a docker repository at doker hub in my account before merging the branches, This created repository/ image name I add in my .yml file of project. Now I can successfully merge the two branches according to above-mentioned steps.
