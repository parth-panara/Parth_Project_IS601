Encapsulation

It means to protect(encapsulate) all the data and functions into class. These protected members of class are not accessible from outside the class. We can access it within a class. Below is the example of calculator project which demonstrates the encapsulation.


image(a) Calculator program demonstrates Encapsulation


In this above image of the calculator program has four methods namely add, subtract, multiply, division.

Calculator holds the instance . Inheritance

This principle takes place in parent class(base class) and child class(subclass).


image(b) Calculator program demonstrates Inheritance


In calculator program, the class Calculator called parent class. The class Addition(), class Subtraction(), class Multiplication(), and class Division() are all child classes of class Calculation() as shown in the image(b). Here, the subclasses inherit the property of its parent class. This demonstrates Inheritance.

Calculator class has methods named convert_args_to_tuple_of_float, and creat method in respectively line number 11 and 14 of image (c). Due to having this property in base class Calculator, the all subclasses Addition(), Subtraction(), Multiplication(), and Division() are able to inherit it for its property.

Polymorphism

Polymorphism means having vivid or different forms.

Polymorphism refers to the ability of the function with the same name to carry different functionality altogether. It creates a structure that can use many forms of objects. Image(d) illustrates the principle of polymorphism in calculator program.


image(d) Calculator program demonstrates Polymorphism


In calculator program, get_result() method of line number 11, 17, 23, 29 convert on any object of method Add(), Subtract(), Multiply(), and Divide().

So, we can call get_result() method on any class of calculation (Addition(), Subtraction(), Multiplication(), and Division() classes ) and we will get a result. Thus, the overridden of get_result() method in classes present a polymorphism concept.

Abstraction

The abstraction principle helps to hide all relevant data about object(calculation) in order to reduce complexity and increase efficiency.

get_result() is performing an addition method in line number 35 of image(c or e). It's simply calling a loop in addition class of calculator. In the loop, we have an add function which abstracting a process of doing addition in a list of numbers.


image(e) Highlighted portion from image c


we have one function as a list and another is add() in a loop of def get_result(self).

add(value, sum_of_values) function calls the addition calculation method. Both function will give a sum of list of numbers. Thus, More complex of lists are broken down in one the function to another function and that called Abstraction.

This entire process of code will be hidden from user and only will invoke by on classes Addition() Subtraction(), Multiplication(), Division(). User is only allowed to run these methods according to their task.
